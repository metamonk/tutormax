# TutorMax Frontend Redesign PRD

## Overview
Complete frontend redesign with modern fintech aesthetic while preserving all existing functionality and data flow.

## Design Philosophy
- Modern fintech aesthetic (trust-building colors, professional polish)
- Cohesive visual language across all pages and components
- Dark and light mode support
- Improved user experience through better layout and visual hierarchy
- NO functionality changes - only visual/layout improvements

## Workflow for Each Component/Page
1. Generate initial design/code with v0 MCP (rapid prototyping)
2. Validate accessibility and patterns with Kibo UI MCP
3. Integrate with existing features and functionality
4. Test and refine

## Design System Foundation

### Theme Implementation
- Apply new OKLCH-based color system from v0
- Professional blue primary (hue 250) for trust and stability
- Success green (hue 145) for positive metrics
- Warning amber (hue 65) for alerts
- Destructive red (hue 25) for critical issues
- Blue-tinted neutral grays for cohesion
- 5 distinct chart colors for data visualization

### Typography & Spacing
- Maintain Geist font family (already modern and clean)
- Improve typographic hierarchy
- Update spacing scale for better rhythm
- Enhance readability on mobile and desktop

### Component System
- Refactor existing shadcn/ui components with new theme
- Ensure consistency across all component variants
- Add smooth transitions and micro-interactions
- Validate accessibility with Kibo UI patterns

## Technical Constraints
- NO breaking changes to existing functionality
- NO changes to API calls or data flow
- NO changes to state management logic
- Must maintain PWA compatibility
- Must maintain all existing features
- Must work with existing authentication system
- Must work with existing WebSocket integration

## Success Criteria
- Professional fintech aesthetic throughout
- Cohesive visual language
- Dark/light mode fully functional
- All existing features work identically
- Improved user experience metrics
- WCAG AA accessibility compliance
- No performance regressions
- Positive user feedback on new design
